    SMPL + MANO Research

    Center of Vision, Cognition, Learning, and Autonomy at UCLA

    In summer of 2020, I was a research assistant at the Center of Vision, Cognition, Learning, and Autonomy at UCLA, working under Tengyu Liu. Over the summer, I worked with combining SMPL (body) and MANO (hand) models, utilizing Python libraries such as Tensorflow and Numpy to predict the entire body's mesh using a list of vertices.

    The image above shows a part of the process. The image on the left shows an example of the first part of the process, which was transforming the hand to the right position and angle given the vertices of the body using SMPL. Then, I would figure out the individual transformations for each joint in the hand model, as seen on the right. Finally, I would connect the two, and run the model to get the final surface of the body and hand.

    Languages/Technologies Used: Python, PyTorch, TensorFlow

    Coding Homies Community

    April - June 2021

    For our capstone class, my team and I had to create an application using C++ and the boost library, utilizing NginxConfig to declare handlers and creating a complex request handler that responded to various requests. Our final project, Coding Homies Community, was a site where users could create topics and write posts underneath the topics, and other users could add on by adding posts to each topic.

    The project included a robust API with the paths defined in a NginxConfig file, which was interpreted by a parser. The corresponding request parser would handle the request appropriately, sending a boost::beast::http::response object. In addition to source code, our team wrote thorough unit and integration tests. Our final project was deployed onto Google Cloud Platform by utilizing Docker containers.


    Languages and Technologies Used: C++, SQLite, CMake, Docker, Google Cloud Platform, NginxConfig

    Benky

    June 2020 - September 2020

    For my summer personal project, I built a complex flashcard site where the use can create a set of terms, practice by flipping through flashcards, and then completing a test. The project was built with React.JS, Node.JS, Firebase, and MongoDB.

    The user can create a new set, where he/she will define the title and the terms and definitions of their items. The title, terms, and definitions can be edited if necessary.

    The user can then practice the set using a online flashcard-like system. This can save paper and can help create a pleasant studying experience.

    The user can finally test their knowledge by ensuring that they can connect a particular term to its correct definition. After taking the test, the user can view their results, find out their mistakes, and continue practicing or retake the test.

    In addition to the mentioned features, benky uses authentification - thus, sets and test results are connected to an email. Authentification for this application is managed using Firebase.

    Languages and Technologies Used: React.JS, Material UI, Node.JS, Express.JS, MongoDB, Firebase
